1. Project Overview

This dashboard provides an interactive overview of the Software Requirements Specification (SRS) for Project Azaadi. The project's primary goal is to achieve true platform independence for the C programming language. We will accomplish this by developing a custom compiler to translate C code into a proprietary, platform-agnostic bytecode, and a custom virtual machine (VM) to execute that bytecode on any supported operating system. This approach, inspired by the Java ecosystem, will allow developers to "write once, run anywhere."

2. System Architecture

The Azaadi ecosystem is composed of two core components: the Compiler and the Virtual Machine (VM). The Compiler handles the translation of human-readable C code into machine-executable bytecode, while the VM provides the runtime environment to execute this bytecode. 3. Key Requirements

The project is guided by specific technical and hardware requirements. The following sections outline the necessary hardware to run the Azaadi VM and the operating systems we are committed to supporting. Click each category to expand and view the details.

  • CPU: Dual-core processor (@ >2GHz)
  • RAM: 4 GB or greater
  • Storage: 50 GB free space
  • Windows 10 and later
  • macOS 11 (Big Sur) and later
  • Linux (Ubuntu 20.04+, Fedora 34+)
